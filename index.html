Jira Board: Workflow Triggers

1
Branch Created → Move to In Progress
When a developer creates a branch linked to a ticket, it signals active work has begun. The ticket automatically moves from Next Up to In Progress.
NEXT UP
→
IN PROGRESS
Only triggers if the ticket is currently in "Next Up"
Branch
Created
2
Pull Request Created → Move to In Review
When a developer opens a pull request, the code is ready for peer review. The ticket automatically moves from In Progress to In Review.
IN PROGRESS
→
IN REVIEW
Only triggers if the ticket is currently in "In Progress"
Pull Request
Created
3
Pull Request Merged → Move to Done
When a pull request is merged, the work is complete and in the codebase. The ticket automatically moves to Done regardless of whether it was in In Review or QA.
IN REVIEW
→
DONE
QA
→
DONE
Triggers from both "In Review" and "QA" statuses

Naming Convention
Include the ticket key in your branch, commit, and PR for triggers to work.
Branch
PD-{ticket-key}/short-description
Example
PD-795/fix-claims-ui
Commit
[PD-{ticket-key}] short description
Example
[PD-795] fix success message after claim submission
Pull Request
[PD-{ticket-key}] short description
Example
[PD-795] Fix claims UI — success state feedback
